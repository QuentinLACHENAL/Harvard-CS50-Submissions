My CS50 Journey: The Good, the Bad, and the Segfaults
This repository is a personal archive of my progress through Harvard's CS50x and CS50p. It’s not just a collection of code; it’s a record of the hours spent staring at a terminal, talking to a rubber duck, and eventually finding that one missing semicolon that broke everything.

The Two Paths
CS50x: Introduction to Computer Science
The "flagship" course. It doesn't just teach you how to code; it teaches you how computers actually work under the hood.

The Experience: It starts with the absolute basics in Scratch but quickly moves into the "C era." Dealing with manual memory management and pointers was a brutal but necessary rite of passage. By the end, you transition into higher-level languages like Python and SQL, building full-stack web applications.

Core Topics: Low-level memory, Algorithms (Big O), Data Structures (Linked Lists, Hash Tables), and Web Development.

CS50p: Introduction to Programming with Python
A deep dive into the world of Python. If CS50x is about the "how," CS50p is about the "do."

The Experience: This course focuses on writing clean, efficient, and "Pythonic" code. It covers the stuff you actually use in the real world: regular expressions, working with APIs, unit testing, and Object-Oriented Programming.

Core Topics: Libraries, File I/O, Regular Expressions (Regex), and Object-Oriented Programming (OOP).

Tech Stack & Tools
Languages: C, Python, SQL, JavaScript, HTML, CSS.

Frameworks: Flask (for web apps) and Jinja.

Testing & Debugging: Valgrind (the memory leak hunter), GDB, and Pytest.

The Environment: VS Code via the CS50 Codespace (GitHub-hosted).

The Reality Check: Difficulty
If you’re looking at this repo and thinking about starting the courses yourself, here is the honest breakdown:

CS50x Difficulty: 9/10. The learning curve isn't a slope; it's a cliff. Week 3 (Algorithms) and Week 5 (Data Structures) are notorious for a reason. But finishing them makes you feel invincible.

CS50p Difficulty: 6/10. It’s still challenging, but since Python handles the "boring" stuff (like memory) for you, you can focus more on logic and building cool things quickly.

Why this Repo exists
I kept this repo to track my growth as a developer. You’ll find my solutions to the Problem Sets (PSETs) and my final projects.

A note on Academic Honesty: If you are currently taking the course, please don't just copy-paste. The struggle is where the actual learning happens. Use these files for inspiration or to get unstuck, but write your own logic. Your brain will thank you later.
